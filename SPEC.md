# Winny 記帳小幫手 — 規格書

## 1. 產品概述

「Winny 記帳小幫手」是一個串接 LINE 官方帳號的日常記帳工具。使用者在 LINE 聊天室裡用一句自然語言輸入支出或收入，機器人會自動解析金額、項目與分類，寫入 Google Sheets 作為資料庫；使用者也可隨時查詢週／月／季／年的消費統計。

- **LINE 官方帳號**：Winny 記帳小幫手（Basic ID：`@844nkred`）
- **資料儲存**：Google Sheets（單一工作表，分頁名稱可自訂，預設「記帳」）
- **執行環境**：Node.js + Express，本機開發時透過 ngrok 對外提供 Webhook

## 2. 目標與非目標

**目標**
- 用最低操作成本（一句話）完成記帳
- 分類判斷盡量自動化，且能隨使用者的歷史紀錄持續變準
- 資料存放在使用者自己的 Google Sheet，可隨時用 Sheets 介面手動檢視、匯出、繪圖
- 支援基本的統計需求（週／月／季／年）

**非目標（目前不處理）**
- 多人共同記帳、群組分帳
- 圖片/收據辨識（OCR）記帳
- 記帳提醒、推播通知
- 預算設定與超支警示
- 正式的多使用者後台或帳號系統（目前以 LINE `userId` 區分資料列即可）

## 3. 使用者互動流程

### 3.1 記帳（單筆）

輸入範例：
```
早餐 60
60 早餐
加油 300 #交通
+5000 薪水
```

規則：
- 訊息開頭為 `+` 視為「收入」，其餘視為「支出」
- 文字中第一個出現的數字視為金額（可為小數）
- 訊息中若含 `#關鍵字`，該關鍵字視為手動指定分類，並從文字中移除
- 移除金額與分類標籤後剩餘文字視為「項目」名稱；若無剩餘文字則標記為「未命名項目」
- 無手動分類時，依序用「歷史同名項目分類」→「內建關鍵字」判斷分類，最終仍無法判斷則歸類「其他」

回覆範例：
```
✅ 已記錄 支出：餐飲 - 早餐 60 元
💰 已記錄 收入：其他 - 薪水 5000 元
```

### 3.2 記帳（多筆／一行一筆）

使用者可在同一則 LINE 訊息中換行輸入多筆：
```
高麗菜 140
紅蘿蔔 200
全聯 770
```
機器人會逐行各自解析、各自寫入一列資料，並在單一回覆中列出所有成功記錄的項目；無法解析的行會列在回覆最後方並標記「看不懂」，不會寫入資料。

### 3.3 分類自動學習

每次記帳前，機器人會即時讀取該使用者過去所有支出紀錄，統計「項目名稱 → 分類」的出現次數，取出現次數最多的分類作為預設值（多數決）。

- 資料來源不限於 LINE 對話：使用者直接在 Google Sheet 網頁上手動修改「分類」欄位，一樣會被下次查詢納入統計，效果等同於用 `#分類` 標籤過
- 若同一項目名稱历史上出現多種分類，取次數最高者；使用者可持續用新分類覆蓋，多數決會隨紀錄增加而改變

### 3.4 統計查詢

| 使用者輸入 | 對應區間 |
|---|---|
| `本週` / `週統計` / `這週` | 本週（週一至週日，Asia/Taipei 時區） |
| `本月` / `月統計` / `統計` / `查詢` | 本月（自然月） |
| `本季` / `季統計` | 本季（Q1: 1–3月、Q2: 4–6月、Q3: 7–9月、Q4: 10–12月） |
| `本年` / `年統計` / `今年` | 本年度（自然年） |

回覆內容包含：
- 該區間標籤（例如「2026-07」「2026 Q3」「2026-07-06 ~ 2026-07-12」）
- 各分類的支出小計（由高到低排序）
- 支出總計
- 若該區間有收入紀錄，額外顯示收入總計與結餘（收入 − 支出）

### 3.5 刪除紀錄

輸入 `刪除` / `刪除上一筆` / `undo`：刪除**該使用者**在 Google Sheet 中「最後一筆」資料列（依 UserId 篩選，非全表最後一列）。找不到可刪除的紀錄時回覆提示訊息。

> 目前僅支援刪除最後一筆，無法指定刪除某一筆歷史紀錄或整批刪除。

### 3.6 說明

輸入 `說明` / `help` / `幫助`：回覆完整使用說明文字。

## 4. 分類體系

### 4.1 內建關鍵字分類（`lib/parser.js`）

| 分類 | 關鍵字範例 |
|---|---|
| 餐飲 | 早餐、午餐、晚餐、消夜、宵夜、飲料、咖啡、便當、聚餐、零食、手搖、吃、喝 |
| 交通 | 公車、捷運、計程車、uber、加油、停車、高鐵、火車、機票、悠遊卡、油錢 |
| 購物 | 衣服、鞋、包包、購物、3c、電器、網購 |
| 娛樂 | 電影、遊戲、唱歌、ktv、旅遊、展覽、訂閱、netflix、門票 |
| 居家 | 房租、水電、瓦斯、網路費、管理費、家具、日用品 |
| 醫療 | 醫院、藥、看診、牙醫、健檢、掛號 |
| 其他 | 上述皆未命中時的預設值 |

判斷方式為關鍵字「包含比對」（不分大小寫），依表格順序找到第一個命中的分類即採用。

### 4.2 分類優先順序（由高到低）

1. 訊息中明確的 `#分類` 標籤
2. 該使用者歷史紀錄中，同名項目出現次數最多的分類（見 3.3）
3. 內建關鍵字比對（見 4.1）
4. 皆未命中 → 「其他」

## 5. 資料模型

### 5.1 Google Sheet 欄位（工作表分頁，預設「記帳」）

| 欄位 | 內容 | 範例 |
|---|---|---|
| A 日期時間 | 記帳當下的 Asia/Taipei 時間，格式 `YYYY-MM-DD HH:mm:ss` | `2026-07-11 11:29:03` |
| B 類型 | `支出` 或 `收入` | `支出` |
| C 分類 | 中文分類名稱 | `餐飲` |
| D 項目 | 使用者輸入的項目文字（可能為使用者自訂中文詞） | `早餐` |
| E 金額 | 數字（可含小數） | `100` |
| F UserId | LINE 使用者的內部 ID（用於區分不同使用者的資料與統計範圍） | `U4af...` |

- 首列（Header）由程式自動偵測並補齊，不需手動建立
- 新資料一律以「附加（append）」方式寫入表尾，不做排序

### 5.2 資料存取邏輯（`lib/sheets.js`）

| 函式 | 用途 |
|---|---|
| `appendExpense(entry)` | 寫入一筆新紀錄，附加 Asia/Taipei 時間戳 |
| `getSummary(userId, period, referenceDate?)` | 依 `week/month/quarter/year` 計算日期區間，彙總該使用者在區間內的分類小計與收支總計 |
| `getItemCategoryHistory(userId)` | 讀取該使用者全部支出紀錄，回傳「項目 → 最常見分類」的對照表 |
| `deleteLastExpense(userId)` | 找出該使用者最後一筆資料列並刪除（使用 Sheets API `batchUpdate` 的 `deleteDimension`） |

日期區間計算一律以 `Asia/Taipei` 時區為準，內部先轉換成 `YYYY-MM-DD` 字串再做字典序比較，避免不同語系/格式造成比對失誤。

## 6. 系統架構

```
LINE 使用者
   │  傳送文字訊息
   ▼
LINE Messaging API（Winny 記帳小幫手 Channel）
   │  Webhook (HTTPS POST) → https://line-winny-line-bot.onrender.com/webhook
   ▼
Render Web Service（Free 方案，24 小時常駐，閒置後會休眠）
   │
   ▼
Express 伺服器（index.js，port 3000）
   ├─ lib/parser.js   解析訊息文字 → {type, category, item, amount}
   └─ lib/sheets.js   讀寫 Google Sheets（服務帳戶驗證）
   │
   ▼
Google Sheets API
   │
   ▼
使用者的 Google Sheet（記帳資料庫）
```

正式環境部署在 Render，程式碼庫為 GitHub `Winny1020/line--`（`main` 分支），推送新 commit 會觸發 Render 自動重新部署。本機開發時可另外用 ngrok 暫時對外提供服務，僅供除錯，不影響正式環境（見 README「四、本機開發與除錯」）。

## 7. 外部服務與憑證

| 服務 | 用途 | 對應環境變數 |
|---|---|---|
| LINE Messaging API | 接收使用者訊息、回覆訊息 | `LINE_CHANNEL_ACCESS_TOKEN`、`LINE_CHANNEL_SECRET` |
| Google Sheets API | 讀寫記帳資料 | `GOOGLE_SERVICE_ACCOUNT_EMAIL`、`GOOGLE_PRIVATE_KEY`、`GOOGLE_SHEET_ID`、`GOOGLE_SHEET_NAME` |
| Render | 正式環境託管（Web Service，Free 方案） | 於 Render Dashboard 的 Environment Variables 設定，內容與 `.env` 相同 |
| GitHub | 程式碼版本控制、觸發 Render 自動部署 | 無（透過 SSH 金鑰推送） |

- Google 端採用服務帳戶（Service Account）驗證，需將目標 Google Sheet 共用給該服務帳戶 email 並給予「編輯者」權限
- 本機開發用的憑證存放於專案根目錄的 `.env`（已被 `.gitignore` 排除，不會進版控）；正式環境的同一組憑證另外設定在 Render 的 Environment Variables

## 10. 已知限制

- 目前僅支援單一使用者情境下的「刪除最後一筆」，不支援刪除任意歷史紀錄或批次刪除
- 每次記帳或查詢都會讀取整份 Google Sheet 做記憶體運算，資料量極大時（數萬列以上）效能會下降
- 分類學習僅採「多數決」，不會理解語意相近但用詞不同的項目（例如「拿鐵」與「咖啡」視為不同項目）
- Render Free 方案閒置一段時間會自動休眠，休眠後第一則訊息可能延遲最多 50 秒才回覆；需要秒回須升級付費方案
